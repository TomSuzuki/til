# HTTP ステータスコード
- 情報レスポンス (1XX)
- 成功レスポンス (2XX)
- リダイレクト (3XX)
- クライアントエラー (4XX)
- サーバエラー (5XX)

## 情報レスポンス
### 100 Continue
無視していい感じのステータスコード。暫定のレスポンスでその時点では問題が発生していない。

## 成功レスポンス
### 200 OK
リクエストが成功した。メソッドによって若干意味が異なる。
- `GET` リソースが読み込まれ、メッセージ本文で転送された。
- `HEAD` メッセージ本文にエンティティヘッダーある。
- `PUT` または POST: 操作の結果を表すリソースがメッセージ本文で送信される。
- `TRACE` メッセージ本文に、サーバーが受け取ったリクエストメッセージが含まれている。

### 201 Created
リクエストが成功し、新たなリソースが作成されたことを示す。`POST`とか`PUT`とかのレスポンス。

## リダイレクションメッセージ
### 303 See Other
リクエストされたリソースを別のURIで`GET`リクエストを使用して取得すようクライアントを誘導するために送信する。

## クライアントエラーレスポンス
### 400 Bad Request
構文エラー。サーバーがリクエストを理解できない。

### 401 Unauthorized
許可なし。認証してね。

### 404 Not Found
見つからないよ。

### 429 Too Many Requests
大量のリクエストを送ったためリクエスト回数の制限に達しました。

## サーバーエラーレスポンス
### 500 Internal Server Error
サーバーで何かしらエラーが発生。

### 503 Service Unavailable
サーバーダウンの可能性。
